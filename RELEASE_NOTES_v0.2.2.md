# Release v0.2.2 - 管理控制台与隐私过滤大升级

本次更新聚焦于管理控制台能力扩展、用户与隐私管理的批量化和精细化，以及对版本更新、系统设置等多处界面体验的统一与打磨。

## 🎉 主要新功能

### 管理控制台

- **最近可用性网格**：新增 GitHub 风格的 30 天服务可用性网格，每天一格，颜色深浅直观反映当日可用率，鼠标悬停显示具体日期、可用率与检测次数，并汇总近 30 天平均可用率与总检测次数。
- **自定义控制台卡片**：管理控制台现在支持自定义要显示的数据分析卡片，可在「自定义卡片」面板中勾选启用、拖拽排序。可选卡片包括：最近可用性、并发概览、切换速率趋势、吞吐量趋势、延迟分布、错误分布、错误趋势、OpenAI Token 统计、告警事件、系统日志、模型分布、Token 使用趋势、今日请求量、总 Token、平均响应速度等。布局会持久化到本地，刷新后保持。

### 用户管理

- **多选与批量修改**：用户列表新增多选能力，支持全选当前页、批量修改（状态 / 并发数 / 备注 / 分组，可按字段勾选是否修改）、批量删除，并显示选中数量与部分成功/失败统计。

### 顶部菜单栏

- **全局时间范围按钮**：仪表盘与管理控制台页面的「时间范围 / 颗粒度」调节条已迁移至顶部菜单栏的图标按钮（位于通知按钮左侧），点击弹出弹窗统一修改日期范围与粒度，选择会全局共享并持久化。

### 版本更新弹窗

- **富文本 / Markdown 渲染**：升级内容弹窗现在支持 Markdown 与富文本（GitHub Flavored Markdown），列表、代码块、表格、链接等均可正常排版，不再堆成一团。
- **撤销固定分类**：移除原先固定的「主要功能 / 新增功能 / 迁移 / 修复」四个分类，直接按发布正文原样呈现。
- **新增操作按钮**：弹窗右下角新增「关闭」与「开始升级」两个按钮，可直接从弹窗触发升级。

### 隐私过滤

- **应用对象**：可选择隐私过滤生效的用户范围 —— 全部用户 / 部分用户（支持搜索选择）/ 仅管理员。
- **渠道维度**：可选择过滤生效的渠道维度 —— 全部渠道 / 按分组 / 按渠道 / 按账号。
- **扩充内置规则**：新增 IPv6、JWT、PEM 私钥、AWS Access Key、GitHub PAT、Slack Token、信用卡号、中国车牌号、URL 敏感查询参数等内置脱敏规则。
- **界面优化**：保存按钮移至顶部并回归标准尺寸；「添加规则」按钮修复过大问题；全局开关组件开启态颜色加深，明暗模式下都更醒目。

### 模块市场

- **内置功能卡片**：原「系统设置 → 功能开关」中的频道监控、可用频道、风控中心、隐私过滤、邀请返利已迁移为模块市场顶部的「内置功能」卡片，可直接启用/禁用并跳转配置。

## ✨ 改进

### 账号管理

- 账号列表「自动刷新」按钮高度已与「刷新」按钮对齐。
- 「导入」「导出」按钮从「更多操作」下拉中移出，作为独立按钮展示，风格与整体一致。
- 账号筛选器平台与类型联动：移除「全部平台」中已失效的 Anti-Gravity 选项；类型按所选平台动态显示（Anthropic / OpenAI / Gemini / Custom 各自只显示支持的类型）。
- 添加账号时 Gemini + AI Studio 反代不再误显示 OAuth / API Key / Vertex 选项；AI Studio 反代模式下隐藏不适用 的池模式；修复选择 AI Studio 反代后顶部步骤标题与第二步标题消失的 Bug。
- Custom 模型第二步标题修正为「Custom 账号添加」。
- 无论选择哪个平台，过期时间、并发数、负载因子、优先级、计费倍率、代理等字段统一整合进可折叠的「高级设置」菜单。

### 系统设置

- 横向 Tab 栏重做为与最新 LightBridge 风格一致的 segmented 控件，移除旧的红→蓝渐变下划线。

### 界面清理

- 移除未上线的「LightBridge Proxy」页面入口（路由与侧边栏），相关源码保留以便后续恢复。

## 🔧 技术变更

- 后端新增 `POST /admin/users/batch-update` 批量更新用户接口（状态 / 并发数 / 备注 / 分组）。
- 后端新增 `GET /admin/channel-monitors/availability?days=30` 近 N 天每日可用率聚合接口（基于 channel monitor daily rollups）。
- 隐私过滤配置结构扩展 `target_scope` / `target_user_ids` / `channel_scope` / `channel_ids` / `account_ids` 字段。
- 前端新增 `useTimeRangeStore`（全局时间范围 store）、`useOpsConsoleLayout`（控制台卡片布局 composable）。
- 版本号升至 **0.2.2**。

## ⚠️ 升级说明

- 本次为前端 + 后端联合更新，请同时升级两者。
- 隐私过滤新增字段对旧配置向后兼容：未设置时默认「全部用户 / 全部渠道」。
- 「最近可用性」网格的数据源为渠道监控（channel monitor）的每日汇总，需开启渠道监控并有检测数据才能显示真实可用率；未开启时网格为空（灰色），不影响其它功能。
