# Release v0.2.22-preview - 模型目录与路由限制统一预览版

本次 Preview 聚焦模型目录、账号模型设置和跨平台分组路由的统一改造。默认行为改为尽可能透传请求：分组只作为账号集合使用，模型列表只作为展示与可选限制依据，不再因为分组平台、渠道平台或旧白名单配置不匹配导致无法路由。

## 新增功能

1. 新增模型目录 / 模型广场
    - 新增账号级模型目录持久化表，用于统一保存上游同步模型、使用方式、来源、同步批次与错误状态。
    - 新增管理员模型目录接口与页面，可按合并、分组、渠道、账号视图查看模型。
    - 新增用户模型目录接口与页面，仅展示模型名称、分组、价格区间、来源数量等实体信息，不暴露具体账号与渠道来源。
    - 重复模型会在合并视图中去重聚合，并展示来源数量与价格区间。

2. 上游模型同步持久化
    - `POST /admin/accounts/:id/models/sync-upstream` 现在会把上游模型写入模型目录，并返回同步状态。
    - `GET /admin/accounts/:id/models` 优先读取当前模型目录，供账号编辑与测试连接使用。
    - OAuth 账号在授权、重新授权、刷新 token 后会最佳努力自动同步模型；同步失败不阻塞账号保存。

3. 账号模型设置重构
    - 原模型列表位置改为模型列表维护与“拉取上游模型”入口。
    - OAuth 账号不再要求手动维护模型列表，改为展示自动同步状态与重试入口。
    - 增加账号级“仅允许模型列表中的请求”开关，默认关闭。
    - 模型映射移动为高级改名能力，只负责“请求模型 -> 上游模型”的映射。

## 修复问题

1. 修复跨平台分组无法路由的问题
    - 请求某个分组时，只要分组内存在可调度账号能够处理对应协议，就不再因为分组平台或渠道平台不一致直接失败。
    - mimo 请求走 cc 平台等跨平台分组场景默认不再被旧模型白名单或平台字段挡住。

2. 移除模型白名单对默认路由的硬拦截
    - `model_mapping` 不再作为默认模型白名单使用。
    - 渠道 `restrict_models` 不再参与网关硬拦截，仅保留历史兼容字段。
    - 未知模型在账号限制开关关闭时会继续透传到上游。

3. 清理旧自映射语义
    - 旧 `model_mapping` 中的精确自映射迁移为模型目录数据。
    - 非自映射继续保留为高级模型映射。
    - 前端批量编辑与账号编辑测试已同步新的模型列表语义。

## 优化调整

1. 模型目录展示
    - 管理员来源明细包含账号、平台、渠道与同步状态。
    - 用户端隐藏具体账号与渠道，仅保留可公开的模型实体信息。
    - 测试连接弹窗模型下拉改为使用当前模型目录；目录为空时仍允许手动输入未知模型测试。

2. 文案统一
    - 清理账号、渠道、模型相关“模型白名单”措辞，统一为“模型列表”“模型范围”“仅允许列表模型”。
    - 同步更新中英文 i18n 文案。

3. 安装与部署辅助
    - 纳入新的 LightBridge 安装管理脚本与本地计划文档，便于后续部署和协作追踪。

## 兼容性 / 破坏性变更

1. 模型限制语义变更
    - 影响范围：依赖旧 `model_mapping` 自映射作为模型白名单的账号。
    - 处理方式：旧自映射会迁移到模型目录；如确实需要拦截未知模型，请在账号上开启“仅允许模型列表中的请求”。

2. 数据库迁移
    - 影响范围：升级后会新增 `account_model_catalog` 表并迁移旧自映射数据。
    - 处理方式：正常执行后端迁移即可；OAuth 自动同步失败只影响模型目录新鲜度，不影响默认透传路由。
