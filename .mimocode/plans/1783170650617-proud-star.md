# Plan: 移除 Antigravity 字样 + 为 Custom 平台实现拉取上游模型

## 任务概述

两个任务：
1. **Antigravity 并入 Gemini**：所有用户可见的 "Antigravity" 字样替换为 "Gemini"，颜色方案统一到 Gemini（蓝色），图标使用 Gemini 星形图标。后端 `AccountPlatform` 类型中的 `'antigravity'` 保留不动（API 兼容），只改前端展示层。
2. **Custom 平台拉取上游模型**：在 `ModelWhitelistSelector.vue` 的 `upstreamSyncPlatforms` 中添加 `'custom'`，使 Custom 平台账号编辑时出现"拉取上游模型"按钮。后端已支持（`buildUpstreamModelsRequest` 中 `IsOpenAI()`/`IsAnthropic()`/`IsGemini()` 等方法对 Custom 平台按协议匹配），无需后端改动。

## 任务二：Custom 平台拉取上游模型

### 文件修改

#### 1. `frontend/src/components/account/ModelWhitelistSelector.vue`
- Line 177: `const upstreamSyncPlatforms = new Set(['anthropic', 'openai', 'gemini', 'antigravity'])` → 添加 `'custom'`
- 即：`const upstreamSyncPlatforms = new Set(['anthropic', 'openai', 'gemini', 'antigravity', 'custom'])`

### 为什么只需要改这一处
- 后端 `FetchUpstreamSupportedModels` 已通过 `IsOpenAI()`/`IsAnthropic()`/`IsGemini()` 等方法支持 Custom 平台（根据 Custom 账号的 protocol 匹配到对应构建器）
- Custom 账号存储了 `base_url` 和 `api_key`，构建器能正确获取这些凭据
- 前端 `accountsAPI.syncUpstreamModels()` API 调用已存在
- `canSyncUpstream` computed 在添加 `'custom'` 后会正确返回 `true`

## 验证方式

1. 启动开发服务器 `pnpm dev`，进入前端
2. 账号管理页面：
   - 平台过滤器中不应出现 "Antigravity" 选项
   - 类型过滤器中不应出现 "Antigravity" 类型
   - Antigravity 类型的账号应显示为 "Gemini" 平台标签（蓝色）
   - Antigravity 账号应使用 Gemini 星形图标
3. 编辑一个 Custom 平台的账号 → 模型白名单区域应出现"拉取上游模型"按钮
4. 编辑一个 Anthropic 平台的账号 → 该按钮仍然存在（回归验证）
5. 运行前端测试：`pnpm test` 确保没有破坏现有测试
