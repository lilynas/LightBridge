# 模型列表集成监控状态 + 快速配置监控

## 目标
将渠道监控中的模型监控信息合并到模型列表页面，每个模型卡片直接显示监控状态（在线/降级/故障），管理员可一键为某个模型快速创建监控。

## 设计方案

### 1. 后端：模型目录 API 增加监控数据

**修改文件**: `backend/internal/service/model_catalog.go`

在 `ModelCatalogModel` 结构体中新增监控相关字段：

```go
type ModelCatalogModel struct {
    // ... existing fields ...

    // 新增：监控状态（由 channel_monitors 表按 model name 匹配聚合）
    MonitorID       *int64   `json:"monitor_id,omitempty"`
    MonitorStatus   string   `json:"monitor_status,omitempty"`   // operational/degraded/failed/error
    MonitorLatencyMs *int    `json:"monitor_latency_ms,omitempty"`
    MonitorAvail7d  *float64 `json:"monitor_availability_7d,omitempty"`
}
```

**批量查询逻辑**（在 `ListCatalog` 末尾，`includeSources=false` 时也执行）：

1. 收集所有 unique model IDs
2. 一次查询 `channel_monitors` 表：`WHERE enabled=true AND primary_model IN (...)`
3. 收集匹配到的 monitor IDs
4. 一次批量查 `channel_monitor_histories`（最新状态）+ 一次批量查 `channel_monitor_daily_rollups`（7d 可用率）
5. 按 model name 填充到 `ModelCatalogModel.Monitor*` 字段

**关键约束**：查询失败时仅 log warning，返回零值（不阻断目录渲染）。

### 2. 后端：快速创建监控端点

**修改文件**: `backend/internal/handler/admin/channel_monitor_handler.go`

新增 `POST /admin/channel-monitors/quick` 端点，接收精简参数：

```go
type quickCreateMonitorRequest struct {
    ModelID        string `json:"model_id" binding:"required"`
    Provider       string `json:"provider" binding:"required,oneof=openai anthropic gemini"`
    APIMode        string `json:"api_mode" binding:"omitempty,oneof=chat_completions responses"`
    Endpoint       string `json:"endpoint" binding:"required"`
    APIKey         string `json:"api_key" binding:"required"`
    IntervalSeconds int   `json:"interval_seconds" binding:"omitempty,min=15,max=3600"`
}
```

自动填充：
- `Name` = `"Monitor: {model_id}"`
- `PrimaryModel` = `model_id`
- `Enabled` = `true`
- `IntervalSeconds` 默认 60

复用现有 `ChannelMonitorService.CreateMonitor` 逻辑。

**修改路由注册**: `backend/internal/server/routes/admin.go` 中添加 `quick` 子路由。

### 4. 前端：模型卡片内嵌监控状态

**修改文件**: `frontend/src/components/model-catalog/ModelCatalogPanel.vue`

在每个模型卡片的"使用模式"标签下方、价格信息之前，插入监控状态区域：

```html
<!-- 监控状态区域 -->
<div v-if="model.monitor_status" class="mb-2 flex items-center gap-2">
  <span :class="statusBadgeClass(model.monitor_status)">
    {{ statusLabel(model.monitor_status) }}
  </span>
  <span v-if="model.monitor_availability_7d != null" class="text-xs text-gray-500">
    {{ formatPercent(model.monitor_availability_7d) }} · {{ formatLatency(model.monitor_latency_ms) }}ms
  </span>
</div>

<!-- 管理员：无监控时显示快捷创建按钮 -->
<button
  v-else-if="admin"
  @click="$emit('quickSetup', model)"
  class="mb-2 inline-flex items-center gap-1 text-xs text-gray-400 hover:text-primary-600"
>
  <Icon name="plus" size="sm" />
  {{ t('modelCatalog.setupMonitor') }}
</button>
```

复用已有的 `useChannelMonitorFormat` composable 获取 `statusLabel`、`statusBadgeClass`、`formatPercent`、`formatLatency`。

### 5. 前端：快速配置监控弹窗

**新增文件**: `frontend/src/components/model-catalog/QuickMonitorSetupDialog.vue`

弹窗内容：
- 标题："为 {model_id} 设置监控"
- 表单字段：
  - Endpoint URL（必填，文本框）
  - API Key（必填，密码框）
  - Provider（下拉选择：OpenAI/Anthropic/Gemini，默认根据 model.platform 推断）
  - API Mode（下拉：chat_completions / responses，默认 chat_completions）
  - 检测间隔（下拉：30s/60s/120s/300s，默认 60s）
- 按钮：取消 / 创建并启用

提交后调用 `createQuickMonitor`，成功后刷新模型列表。

### 6. 前端：View 页面事件连接

**修改文件**: `frontend/src/views/admin/ModelCatalogView.vue`

- 监听 `ModelCatalogPanel` 的 `@quickSetup` 事件
- 打开 `QuickMonitorSetupDialog`，传入模型信息
- 创建成功后重新加载目录数据

**修改文件**: `frontend/src/views/user/ModelCatalogView.vue`

- 无需修改（用户视图只显示状态，不显示创建按钮）

## 验证方案

1. 启动本地开发服务器 `npx vite --host`
2. 管理员登录后访问 `/admin/model-catalog`
3. 确认已配置监控的模型显示状态徽章（颜色+文字）、可用率和延迟
4. 确认未配置监控的模型显示"设置监控"链接
5. 点击"设置监控"，填写 endpoint 和 key，提交后确认模型卡片出现监控状态
6. 用户视图 `/model-catalog` 确认只显示状态，无设置按钮
7. 确认渠道监控页面 `/admin/channels/monitor` 中新增的监控正常出现
